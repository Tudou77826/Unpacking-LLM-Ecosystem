# 📚 解构 LLM：完整课程目录

> **课程定位**：面向程序员的 AI 编程公开课——不是教你"怎么用"，而是帮你搞懂"为什么是这样"。

## 🧱 第一章：大模型的基石（共 3 课时）

> **本章目标**：建立对 LLM 工作方式的直觉理解，揭示其"原生困境"，理解为什么后续会出现提示词工程、RAG、Tool Calling、Harness Engineering 等技术——它们都是为了解决这些困境。

---

### 第 1 课：LLM 的本质——概率预测与语义空间

**📝 课程大纲**
- **从扔硬币到语言模型**：概率分布的直觉
- **Token 与 Next Token Prediction**：
  - 文本世界的"原子"
  - "预测下一个词"的数学含义
  - Softmax、采样与 Temperature
- **Embedding 与语义空间**：
  - 为什么文字必须变成向量？
  - 在高维空间"看见"语义
  - "国王-男人+女人≈女王"的几何直觉
- **核心认知**：LLM 不是"思考"，而是"计算概率"

**🔓 揭开的疑惑**
- 为什么说 LLM 本质上是"高级自动补全"？
- 同样的问题为什么每次回答不一样？
- 为什么计算机能理解"苹果"和"香蕉"是类似的东西？
- Temperature=0 和 Temperature=1 到底差在哪？

**🚧 引出的困境**
- 概率生成的不确定性 → **可控性困境**
- 语义理解依赖上下文 → **上下文理解困境**

---

### 第 2 课：Transformer 与训练——能力的诞生

**📝 课程大纲**
- **为什么是 Transformer**：
  - RNN/LSTM 的瓶颈：顺序处理的代价
  - Self-Attention：让每个词"看见"所有词
  - 注意力机制的直觉与影响
- **训练的三部曲**：
  - 预训练：在海量文本上学习"预测下一个词"
  - 微调（SFT）：从"话痨"变成"助手"
  - 对齐（RLHF/DPO）：让模型"听话"
- **涌现与规模**：为什么大模型突然会了小模型不会的事
- **核心认知**：模型的"知识"压缩存储在参数中

**🔓 揭开的疑惑**
- 为什么 Transformer 能完全并行化？
- "注意力"到底在注意什么？
- 为什么只预训练的模型不像"助手"？
- 模型参数里到底"存"了什么？

**🚧 引出的困境**
- 知识在训练时固化 → **知识时效性困境**
- 知识压缩存储 → **知识精确性困境**

### 第 3 课：原生困境——LLM 的边界与解决之路

**📝 课程大纲**
- **困境一：知识固化**
  - 训练截止日期后的新知识无法获取
  - 私有数据、企业内部数据无法进入模型
  - → **解决方向**：RAG（第三章）

- **困境二：上下文有限且易迷失**
  - Context Window 有长度限制
  - "Lost in the Middle"：塞进去的不等于被用到
  - → **解决方向**：上下文工程（第三章）

- **困境三：只会"说"不会"做"**
  - 模型只能输出文本，不能执行操作
  - 无法访问数据库、调用 API、读写文件
  - → **解决方向**：Tool Calling & MCP（第四章）

- **困境四：不可控与幻觉**
  - 概率生成必然伴随幻觉
  - 输出不可预测，行为难以约束
  - → **解决方向**：提示词工程（第二章）+ Harness Engineering（第五章）

- **困境五：无法稳定遵循指令**
  - 同样的提示词，结果不一致
  - 复杂任务容易"跑偏"
  - → **解决方向**：提示词工程（第二章）

- **全景图**：后续章节如何一一解决这些困境

**🔓 揭开的疑惑**
- 为什么模型会"一本正经地胡说八道"？
- 为什么不能把所有知识都"教"给模型？
- 为什么光靠提示词有时不管用？
- 为什么需要 RAG、Tool Calling、Harness 这一套组合拳？

**🎯 本章小结：技术演进的内在逻辑**

| 原生困境 | 解决技术 | 所在章节 |
|----------|----------|----------|
| 无法稳定遵循指令 | Prompt Engineering | 第二章 |
| 知识固化、无法更新 | RAG | 第三章 |
| 上下文有限、易迷失 | Context Engineering | 第三章 |
| 只能输出文本、无法行动 | Tool Calling / MCP | 第四章 |
| 行为不可控、幻觉难消除 | Harness Engineering | 第五章 |
